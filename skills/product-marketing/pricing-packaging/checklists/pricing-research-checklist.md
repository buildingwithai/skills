# Pricing Research Checklist

Complete research checklist before setting prices.

## Phase 1: Foundation Research

### Customer Understanding

- [ ] **Identify target segments** (SMB, mid-market, enterprise)
  - Define segment criteria (company size, industry, use case)
  - Prioritize primary vs secondary segments
  
- [ ] **Map customer personas**
  - Identify decision-makers (who signs off?)
  - Understand budget authority ($X-Y typical budget)
  - Document buying process (self-serve vs sales-led)

- [ ] **Document jobs-to-be-done**
  - What problem does product solve?
  - What's current alternative/workaround?
  - Quantify pain (time wasted, money lost, etc.)

**Status:** Not Started / In Progress / Complete

---

### Willingness to Pay (WTP) Research

- [ ] **Van Westendorp PSM survey** (n=30-50 minimum)
  - Too expensive price point
  - Getting expensive price point
  - Bargain/great value price point
  - Too cheap price point
  - Analyze: Optimal Price Point (OPP), acceptable range

- [ ] **Conjoint analysis** (if budget permits)
  - Test feature combinations
  - Understand feature value weights
  - Determine willingness to pay by feature

- [ ] **Customer interviews** (n=20-30 minimum)
  - Current customers (50%)
  - Qualified prospects (50%)
  - Mix of segments
  - Record quotes, insights

**Deliverable:** WTP research report with price range recommendation

**Status:** Not Started / In Progress / Complete

---

## Phase 2: Competitive Analysis

### Competitor Pricing Audit

- [ ] **Identify top 5 competitors**
  - Direct competitors (same category)
  - Indirect competitors (alternative solutions)
  - Document: company size, funding, market position

- [ ] **Audit competitor pricing models**
  - Pricing model (flat-rate, tiered, per-user, usage, etc.)
  - Value metric (what they charge for)
  - Price points (all tiers)
  - Annual discounts
  - Free trial/freemium offerings

- [ ] **Screenshot competitor pricing pages**
  - Save for reference
  - Note: tier names, messaging, CTAs

- [ ] **Test competitor products** (sign up for trials)
  - Experience onboarding
  - Understand feature gating
  - Note upgrade prompts

- [ ] **Gather customer feedback on competitors**
  - Reviews (G2, Capterra, TrustRadius)
  - Common complaints about pricing
  - Perceived value vs price

**Deliverable:** Competitive pricing audit report

**Status:** Not Started / In Progress / Complete

---

### Market Positioning

- [ ] **Determine competitive positioning**
  - Premium (20%+ above market)
  - Competitive (±10% of market)
  - Value (20%+ below market)

- [ ] **Identify pricing gaps**
  - Unserved price points
  - Feature/price combinations competitors don't offer

**Status:** Not Started / In Progress / Complete

---

## Phase 3: Value Proposition & Differentiation

### Value Quantification

- [ ] **Quantify value delivered**
  - Time saved (hours/week × hourly rate)
  - Revenue generated (sales increase, conversions)
  - Cost reduced (efficiency gains, automation)
  - Risk mitigated (compliance, security)

- [ ] **Build ROI calculator**
  - Input: Customer metrics (team size, volume, etc.)
  - Output: $ value delivered per month/year
  - Use in sales conversations

- [ ] **Document case studies**
  - Customer success stories
  - Quantified outcomes
  - Use in pricing justification

**Deliverable:** Value quantification framework

**Status:** Not Started / In Progress / Complete

---

### Unique Value Drivers

- [ ] **Identify top 3-5 differentiators**
  - Features competitors don't have
  - Better UX, performance, reliability
  - Unique data, insights, integrations

- [ ] **Validate differentiation with customers**
  - Survey: "What makes [Product] different?"
  - Rank differentiators by importance

**Status:** Not Started / In Progress / Complete

---

## Phase 4: Pricing Model Selection

### Model Evaluation

- [ ] **Evaluate pricing model options**
  - Flat-rate: One price, all features
  - Tiered: Good/Better/Best
  - Per-user: $X/user/month
  - Usage-based: $X per API call, GB, etc.
  - Hybrid: Subscription + usage
  - Freemium: Free tier + paid tiers

- [ ] **Select value metric**
  - What do customers pay for?
  - Criteria: correlates with value, easy to understand, grows with customer, measurable
  - Validate with customers (interviews)

- [ ] **Design tier structure** (if tiered)
  - Free tier (if freemium)
  - 3-4 paid tiers (Good/Better/Best)
  - Enterprise tier

**Deliverable:** Pricing model recommendation

**Status:** Not Started / In Progress / Complete

---

## Phase 5: Financial Modeling

### Unit Economics

- [ ] **Calculate target metrics**
  - LTV (Lifetime Value): $_____
  - CAC (Customer Acquisition Cost): $_____
  - LTV:CAC ratio: ___:1 (target 3:1 minimum)
  - Gross margin: ____% (target 80%+ for SaaS)
  - CAC payback: _____ months (target 6-12 months)

- [ ] **Model revenue scenarios**
  - Price point A: Conversion ___%, ARPU $_____, Revenue $_____
  - Price point B: Conversion ___%, ARPU $_____, Revenue $_____
  - Price point C: Conversion ___%, ARPU $_____, Revenue $_____

- [ ] **Sensitivity analysis**
  - If price +10%, revenue impact: _____%
  - If conversion -5%, revenue impact: _____%

**Deliverable:** Financial model with recommended price points

**Status:** Not Started / In Progress / Complete

---

## Phase 6: Packaging Research

### Feature Allocation

- [ ] **List all features**
  - Core features (table stakes)
  - Advanced features (power users)
  - Premium features (enterprise)

- [ ] **Survey customers: Feature value**
  - "Which features are most valuable?" (rank top 5)
  - "Which features would you pay more for?"
  - "What's missing?"

- [ ] **Allocate features to tiers**
  - Free: Core value, limited scale
  - Tier 1: Remove limits, basic features
  - Tier 2: Advanced features, more scale (target 60-70%)
  - Tier 3: Premium features, no limits
  - Enterprise: Custom, security, support

**Deliverable:** Feature-to-tier allocation matrix

**Status:** Not Started / In Progress / Complete

---

## Phase 7: Validation & Testing

### Customer Validation

- [ ] **Present pricing to 10-15 customers**
  - Show mockup pricing page
  - Ask: "Which tier would you choose?"
  - Ask: "Is [price] too high, too low, or about right?"
  - Gather objections, concerns

- [ ] **Test messaging**
  - Annual discount messaging (% vs $ vs months free)
  - Tier descriptions
  - CTA copy

- [ ] **Validate upgrade triggers**
  - When would you upgrade from Free → Paid?
  - When would you upgrade from Tier 1 → Tier 2?

**Deliverable:** Validation report with recommended adjustments

**Status:** Not Started / In Progress / Complete

---

### Internal Alignment

- [ ] **Present to Monetization Council**
  - Product Marketing, Product, Sales, CS, Finance
  - Review research findings
  - Get buy-in on pricing strategy

- [ ] **Sales enablement prep**
  - Draft talk tracks
  - Prepare objection handling
  - Create pricing one-pager

- [ ] **Finance approval**
  - Review revenue projections
  - Approve pricing model
  - Confirm billing system readiness

**Status:** Not Started / In Progress / Complete
