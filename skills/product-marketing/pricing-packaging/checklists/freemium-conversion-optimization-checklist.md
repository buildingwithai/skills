## Phase 1: Onboarding Optimization

### Goal: Get Free Users to "Aha Moment" Faster

- [ ] **Define "aha moment"**
  - What action indicates user "gets it"?
  - Example: Created first project, shared with team, completed workflow
  - Your aha moment: ___________

- [ ] **Measure time to aha**
  - Current: ___% of users reach aha moment within 7 days
  - Target: ___% within 7 days
  - Benchmark: 70%+ is good

- [ ] **Optimize onboarding flow**
  - Remove friction (unnecessary steps, complex setup)
  - Add guidance (tooltips, walkthroughs, progress bar)
  - Personalize (ask use case, customize experience)

- [ ] **Track activation milestones**
  - Milestone 1: ___________
  - Milestone 2: ___________
  - Milestone 3: ___________ (aha moment)
  - % reaching each milestone: ___%, ___%, ___%

**A/B test idea:** Guided setup wizard vs self-serve
**Expected impact:** +10-20% activation rate

**Status:** Not Started / In Progress / Complete

---

### Welcome Email Series

- [ ] **Day 0: Welcome email**
  - Subject: "Welcome to [Product]! Here's how to get started"
  - Content: Quick start guide, first action to take
  - CTA: Complete setup, reach aha moment

- [ ] **Day 3: Tips & tricks**
  - Subject: "[Name], here are 3 ways to get more from [Product]"
  - Content: Power user tips, use cases
  - CTA: Try advanced feature (gated = upgrade prompt)

- [ ] **Day 7: Success story**
  - Subject: "See how [Company] uses [Product]"
  - Content: Case study, quantified results
  - CTA: Unlock [feature] to achieve similar results (upgrade prompt)

- [ ] **Day 14: Upgrade offer**
  - Subject: "Ready for more? Upgrade to Pro"
  - Content: Benefits of paid tier, what you're missing
  - CTA: Start trial or upgrade

**Track:** Open rates, click rates, conversion from each email

**Status:** Not Started / In Progress / Complete

### Usage Limit Optimization

- [ ] **Current limits (free tier):**
  - Limit 1: _____ (e.g., 10 projects)
  - Limit 2: _____ (e.g., 5 users)
  - Limit 3: _____ (e.g., 100 API calls/month)

- [ ] **Analyze limit hit rates**
  - % of free users hitting Limit 1: ___%
  - % of free users hitting Limit 2: ___%
  - % of free users hitting Limit 3: ___%

- [ ] **Conversion when limit hit**
  - Limit 1: ___% convert within 30 days
  - Limit 2: ___% convert within 30 days
  - Limit 3: ___% convert within 30 days

**Insight:** Which limit drives most conversions?

- [ ] **Optimize limits**
  - Too loose (< 20% hit limit): Tighten to create more upgrade triggers
  - Too tight (> 80% hit limit): Loosen to avoid frustrating early users

**A/B test:** Limit at X vs Y (e.g., 5 projects vs 10 projects)
**Measure:** Free-to-paid conversion, user satisfaction

## Phase 3: In-Product Upgrade Prompts

### Goal: Show Value, Not Just "Upgrade"

- [ ] **Audit current prompts**
  - Where do prompts appear? (gated feature click, limit hit, settings)
  - What do they say? (copy)
  - How intrusive? (modal, banner, tooltip)

- [ ] **A/B test prompt copy**
  - **Control:** "Upgrade to Pro to unlock this feature"
  - **Variant:** "Unlock [Feature] to [specific benefit]. Upgrade to Pro."
  - **Expected impact:** +15-25% click-through (value-focused messaging)

- [ ] **Test prompt timing**
  - **Option A:** Show immediately when limit hit (frustrating?)
  - **Option B:** Show after user attempts 2-3 times (demonstrates need)
  - **Option C:** Show after aha moment (experienced value first)

**Optimal:** Option C (after aha moment) = +20-30% conversion

- [ ] **Frequency capping**
  - Max prompts per session: _____
  - Max prompts per week: _____
  - Avoid spam (trains users to ignore)

**Status:** Not Started / In Progress / Complete

---

### Prompt Placement Optimization

- [ ] **High-intent placements** (prioritize these)
  - [ ] Gated feature click (user wants feature now)
  - [ ] Usage limit hit (user needs more capacity)
  - [ ] Settings page (exploring options)
  - [ ] Export/download (extracting value = ready to pay)

- [ ] **Medium-intent placements**
  - [ ] Dashboard (ambient awareness)
  - [ ] Help menu (considering options)

- [ ] **Low-intent placements** (avoid)
  - [ ] Random popups (annoying)
  - [ ] Login screen (too early)

**Status:** Not Started / In Progress / Complete

## Phase 4: Pricing Page & Checkout Optimization

### Goal: Reduce Friction in Purchase Flow

- [ ] **Test pricing page elements**
  - CTA copy: "Get Started" vs "Start Free Trial" vs "Upgrade Now"
  - Tier order: Low-to-high vs high-to-low
  - Highlighted tier: Which tier gets "Most Popular" badge?
  - Annual toggle: Default monthly or annual?

- [ ] **Checkout flow friction audit**
  - Steps required: _____
  - Fields required: _____
  - Time to complete: _____ seconds

- [ ] **Reduce checkout friction**
  - Remove unnecessary fields (email only vs email+name+phone)
  - Auto-fill (pre-populate from free account)
  - Social login (Google, Microsoft)
  - Save payment method (one-click future upgrades)

**Benchmark:** <2 minutes from "Upgrade" click to confirmation

**Status:** Not Started / In Progress / Complete

---

### Trial vs Direct Purchase

- [ ] **Current flow:**
  - Free → Trial (X days) → Paid
  - OR Free → Direct purchase (no trial)

- [ ] **Test trial length** (if offering)
  - 7 days vs 14 days vs 30 days
  - Longer trial = higher conversion (but slower)
  - Benchmark: 14 days standard for B2B SaaS

- [ ] **Credit card requirement**
  - CC required (higher intent, lower volume)
  - No CC required (lower intent, higher volume)
  - Test both, measure net conversions

**Status:** Not Started / In Progress / Complete

## Phase 5: Behavioral Triggers & Campaigns

### Goal: Proactive Outreach at Right Moments

- [ ] **PQL (Product Qualified Lead) definition**
  - Criteria 1: Reached aha moment (yes/no)
  - Criteria 2: Usage ≥ _____ sessions/week
  - Criteria 3: Invited ≥ _____ teammates
  - Criteria 4: Tried gated feature ≥ _____ times

**PQL score:** ___/100 (when > 70 = outreach)

- [ ] **Automated email campaigns for PQLs**
  - Email 1: "You're a power user! Here's what Pro unlocks"
  - Email 2: "Special offer: 20% off first 3 months"
  - Email 3: "Let's chat about your needs" (sales outreach)

- [ ] **In-product messaging for PQLs**
  - Banner: "You're using [Product] a lot! Upgrade for unlimited"
  - Modal: "Unlock [feature] to [benefit]" (personalized)

**Conversion rate:** PQLs convert ___% vs ___% all free users

**Status:** Not Started / In Progress / Complete

## Phase 7: Pricing & Packaging Adjustments

- [ ] **Free tier too generous?** (Check if yes)

**Action:** Tighten limits or gate more features
