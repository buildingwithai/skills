# Pricing Launch Checklist

Complete launch checklist for new or updated pricing.

## Timeline: 4-6 Weeks Pre-Launch

---

## Week 1-2: Pre-Launch Preparation

### Product & Engineering

- [ ] **Feature gating implemented**
  - Tier-based feature flags working
  - Usage limits enforced (API calls, storage, users)
  - Upgrade prompts trigger correctly

- [ ] **Billing system configured**
  - Stripe/Chargebee/billing platform updated
  - Price points configured (monthly, annual)
  - Proration logic tested (upgrades/downgrades)
  - Tax calculation configured (VAT, sales tax)

- [ ] **In-product pricing updated**
  - Settings page shows new tiers/prices
  - Upgrade flow tested (all tiers)
  - Downgrade flow tested
  - Annual vs monthly toggle working

- [ ] **Analytics tracking**
  - Pricing page events tracked
  - Conversion by tier tracked
  - Revenue by tier tracked
  - Upgrade/downgrade events tracked

**Status:** Not Started / In Progress / Complete

---

### Marketing & Website

- [ ] **Pricing page redesigned**
  - New tiers displayed
  - Feature comparison table updated
  - FAQ updated
  - Mobile-responsive
  - Load time <2 seconds

- [ ] **Copy finalized**
  - Tier names and descriptions
  - Feature bullets (5-8 per tier)
  - CTA copy ("Get Started", "Start Free Trial")
  - Annual discount messaging ("Get 2 months free")

- [ ] **Social proof added**
  - Customer logos
  - Reviews/ratings
  - User count ("Trusted by 10,000+")

- [ ] **Blog post drafted**
  - Title: "Introducing New Pricing for [Product]"
  - Sections: Why changing, What's new, Who benefits, FAQ
  - Publish date: Launch day

- [ ] **Landing page A/B test** (optional)
  - Set up experiment (if testing pricing variations)
  - Configure analytics

**Status:** Not Started / In Progress / Complete

---

### Sales & CS Enablement

- [ ] **Sales kickoff completed** (T-2 weeks)
  - Presented: Why changing, what's new, how to position
  - Practiced: Talk tracks, objection handling
  - Distributed: Pricing one-pager, competitive responses

- [ ] **Sales talk tracks documented**
  - Value messaging per tier
  - Competitive positioning
  - Discount policy (authority matrix)

- [ ] **Objection handling playbook**
  - "Too expensive" → ROI response
  - "Competitor cheaper" → Differentiation response
  - "Why price increase?" → Value-add response

- [ ] **CS enablement completed** (T-2 weeks)
  - Expansion playbook (when to upsell)
  - Renewal conversations (price increase handling)
  - Support FAQ prepared

- [ ] **Internal pricing wiki updated**
  - Current pricing (all tiers)
  - Feature matrix
  - Discount policies
  - Talk tracks & objections
  - Process docs (discount approvals, etc.)

**Status:** Not Started / In Progress / Complete

- [ ] **Billing integration tested**
  - Test transactions (all tiers)
  - Verify tax calculation
  - Confirm revenue recognition
  - Test refund/credit process

- [ ] **Reporting dashboards**
  - Conversion by tier
  - MRR/ARR impact

**Status:** Not Started / In Progress / Complete

## Week 3: Customer Communication Prep

### For Price Increases Only

- [ ] **Email drafted: T-60 announcement**
  - Subject, body, CTAs
  - Reviewed by legal (if needed)
  - Approved by CEO/leadership

- [ ] **Email drafted: T-30 reminder**
  - Reminder of change
  - Options (annual discount, downgrade)
  - CTA to discuss

- [ ] **Email drafted: T-7 final reminder**
  - Last chance messaging
  - Clear next steps

- [ ] **In-app notifications**
  - Banner: 60 days notice
  - Modal: 30 days reminder
  - Final: 7 days countdown

- [ ] **FAQ page updated**
  - "Why raising prices?"
  - "When does it take effect?"
  - "Can I keep current price?"
  - "What if I cancel?"

**Status:** Not Started / In Progress / Complete (N/A if new pricing, not increase)

## Week 4: Final Pre-Launch

- [ ] **End-to-end purchase flow tested**

  - Paid tier purchase (monthly)

  - Enterprise contact form

- [ ] **Upgrade/downgrade flows tested**
  - Free → Tier 1
  - Tier 1 → Tier 2

- [ ] **Payment testing**

- [ ] **Tax testing**

  - Reverse charge (B2B)

- [ ] **Mobile testing**

## Launch Day

### Morning (9:00 AM)

- [ ] **Pricing page goes live**
  - Update website
  - Verify: All prices correct, links working
  - Check: Mobile view

- [ ] **Billing system cutover**
  - New prices effective for new customers
  - Existing customers: Grandfathered or migrated (per plan)

- [ ] **Blog post published**
  - Announce new pricing
  - Link from homepage, social media

- [ ] **Internal announcement**
  - Slack/email to all teams: "Pricing is live!"
  - Include: What changed, where to find resources

### Mid-Morning (10:00 AM)

- [ ] **Customer email sent** (if price change)
  - Existing customers notified (if not done T-60)
  - Prospects on trial notified

- [ ] **Sales/CS/Support notified**
  - "Pricing is live, expect questions"
  - Monitor Slack channel for issues

- [ ] **Monitor first transactions**
  - Verify billing working
  - Check for errors
  - Confirm revenue tracking

### Throughout Day

- [ ] **Monitor metrics**
  - Pricing page traffic
  - Conversion rate (by tier)
  - Support ticket volume
  - Social media sentiment

- [ ] **Support team on standby**
  - FAQ ready
  - Escalation path clear
  - Log all issues/questions

- [ ] **Track feedback**
  - Customer comments (email, chat, social)
  - Sales team input (objections)
  - Support tickets (pricing-related)

**Status:** Not Started / In Progress / Complete

## Rollback Plan (If Launch Fails)

### Triggers for Rollback

- Major technical issues (billing broken, wrong charges)
- Massive negative feedback (>50% angry)
- Churn spike (>2x normal in first week)
- Legal/compliance issue

### Rollback Steps

- [ ] **Pause new pricing** (within 24-48 hours)
  - Revert pricing page
  - Stop new sign-ups at new price

- [ ] **Internal announcement**
  - "Pricing change paused while we address feedback"

- [ ] **Fix issues**
  - Refund incorrect charges
  - Address technical bugs
  - Revise messaging

- [ ] **Re-launch or permanent rollback**
  - Re-launch with fixes (if fixable)
  - Permanent rollback to old pricing (if not fixable)

**Status:** Not Applicable / Triggered / Complete
